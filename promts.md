# promts.md

Этот файл нужен для простых запросов, которые можно просто копировать в чат без долгих формулировок.

## Главный промпт

Если не знаешь, что писать, используй это:

```text
Посмотри, на каком этапе сейчас проект linux-helper, и сразу приступай к следующему самому важному шагу. Сначала изучи текущее состояние кода и документов, потом выбери логичное продолжение, реализуй его до конца, добавь или обнови тесты, проверь результат через go test ./... -race и кратко напиши, что именно сделал.
```

## Еще более короткий вариант

```text
Посмотри, где мы остановились в linux-helper, и продолжай работу с самого важного места. Доведи задачу до рабочего состояния и проверь тесты.
```

## Если нужно сначала только понять состояние проекта

```text
Посмотри, на каком этапе сейчас linux-helper. Скажи, что уже готово, что не готово и какой следующий шаг самый правильный. Пока ничего не меняй.
```

## Если нужно продолжать без лишних вопросов

```text
Самостоятельно определи следующий приоритетный шаг для linux-helper и выполни его без лишней теории. Работай в рамках текущей архитектуры проекта, после изменений обнови тесты и проверь go test ./... -race.
```

## Если нужно доделывать проект шаг за шагом

```text
Веди linux-helper дальше шаг за шагом. На каждом сообщении сначала смотри текущее состояние проекта, потом выбирай следующий самый полезный шаг, реализуй его полностью, проверяй тесты и только после этого переходи дальше.
```

## Если задача уже примерно понятна

```text
Продолжай linux-helper в области: <вставь область, например search, tui, recipes, executor>. Сначала быстро изучи текущее состояние этой части, потом доведи следующий логичный кусок работы до конца, добавь или обнови тесты и проверь go test ./... -race.
```

## Самая простая формулировка

```text
Посмотри на каком мы этапе и приступай.
```

## Улучшенная формулировка той же мысли

```text
Посмотри, на каком этапе сейчас проект linux-helper, и приступай к следующему приоритетному шагу. Сначала сам разберись в текущем состоянии проекта, затем реализуй логичное продолжение, не ломая архитектуру, и в конце проверь тесты.
```

## Что лучше использовать

Лучший вариант почти всегда такой:

```text
Посмотри, на каком этапе сейчас проект linux-helper, и сразу приступай к следующему самому важному шагу. Сначала изучи текущее состояние кода и документов, потом выбери логичное продолжение, реализуй его до конца, добавь или обнови тесты, проверь результат через go test ./... -race и кратко напиши, что именно сделал.
```
